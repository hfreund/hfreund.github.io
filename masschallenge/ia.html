MassChallenge

Information Architecture

A ground-up overhaul of a decade old information architecture.

Application overview + condition

Built out of an evolution of a desktop application, this benefits administration webapp for human resource departments and insurance brokers was a challenge to navigate in even the best of conditions. The problems came from a few areas:

  1. Architecture: The system architecture and navigational structure were not designed to accomodate continuous additions of function and modules. Without formal training, new users were often unable to locate even basic tools.
  2. User Interface: Divergent efforts by a series of developers to create new layouts or update UI components resulted in a fractured user experience. Where module archetypes did exist, they tended to be many years out of step with contemporary practices.
  3. Dependencies: Many modules that integrated into the system could not be changed or updated easily due to dependencies or compliance requirements. Therefore, new development was frequently hamstrung to advance in a new direction, which would deviate from the dependent modules.

Working with the development team, I helped create a method for bringing the user interface up to date and designed the future system UI to work towards. The update would take place in two phases:

Phase I

Establish a common denominator for the UI, allowing for system-wide changes in the future.

Phase II

A future plan to progressively transition from the old UI to the new UI. While I was part of the planning for this phase and maintained it as an end goal while working on Phase I, work on Phase II has progressed only through medium-fidelity wireframes.
